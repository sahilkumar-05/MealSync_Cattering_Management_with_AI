## Kya Kahan Deploy Hoga (3 Parts)

| Part | Free Platform | Kyun |
|---|---|---|
| **Database** | Neon (already hai) | Already cloud pe hai, kuch karna nahi |
| **Backend (NestJS)** | Render / Railway | Node.js backend ko continuously chala sakte hain free tier pe |
| **Frontend (React)** | Vercel / Netlify | Static/SPA hosting ke liye best, free aur super easy |

---

## Best Free Combo (Recommended)

### 1. Frontend → **Vercel**
- vercel.com pe GitHub se login karo
- "New Project" → apna GitHub repo select karo → `frontend` folder ko root set karo
- Build command: `npm run build`, Output directory: `dist` (Vite ka default)
- Deploy button dabao — 2 minute mein live URL milega (jaise `mealsync.vercel.app`)
- **Free tier mein koi cold start issue nahi**, kaafi fast hai

### 2. Backend → **Render**
- render.com pe GitHub se login karo
- "New Web Service" → repo select karo → `backend` folder ko root set karo
- Build command: `npm install && npm run build`
- Start command: `npm run start:prod`
- Environment Variables mein wahi `.env` wali values daal do (DATABASE_URL, JWT_SECRET, LLM_API_KEY, wagera)
- Deploy karo — URL milega jaise `mealsync-backend.onrender.com`

**⚠️ Important Free Tier Limitation:** Render ka free tier **15 minutes inactivity ke baad backend so jata hai** (sleep mode). Jab koi request aaye to 30-50 second lagte hain wake up hone mein. Portfolio demo ke liye thoda awkward lag sakta hai (interviewer wait karega).

**Fix/Alternative:** **Railway** try kar sakte ho — usme bhi free tier hai (thoda credit-based, $5/month free credit milta hai jo chhote project ke liye kaafi hota hai) aur sleep issue kam hota hai. Ya phir demo dene se pehle backend URL khud ek baar hit kar lo taake wake ho jaye.

### 3. Database → **Neon**
Kuch karna nahi — already deployed hai cloud pe, bas connection string same rahegi.

## Zaroori Changes Jo Deployment Se Pehle Karne Honge

**1. Frontend mein API URL hardcode mat karo**

`frontend` mein jahan bhi `http://localhost:3000` likha hai, usko environment variable se lena hoga:

File: `frontend/.env`
```
VITE_API_URL=http://localhost:3000
```

Code mein:
```typescript
const API_URL = import.meta.env.VITE_API_URL;
axios.get(`${API_URL}/menus`)
```

Deploy karte waqt Vercel mein Environment Variable set karna:
```
VITE_API_URL=https://mealsync-backend.onrender.com
```

**2. Backend mein CORS sahi origin allow karo**

File: `backend/src/main.ts`
```typescript
app.enableCors({
  origin: ['http://localhost:5173', 'https://mealsync.vercel.app'], // apna Vercel URL yahan daalo
  credentials: true,
});
```
(Localhost bhi rakho taake development mein bhi kaam kare)

**3. Backend `.env` ki values Render/Railway ke dashboard mein manually daalni hongi** (kyunki `.env` file khud GitHub pe push nahi hoti — usko `.gitignore` mein rakhna chahiye already)

**4. `synchronize: true` ko production mein disable karna chahiye** (best practice) — abhi assignment ke liye theek hai, lekin agar chaho to migrations use kar sakte ho baad mein.

---

## LLM (AI Feature) Ka Kya Hoga Deployment Mein?

Jo bhi free LLM (Gemini/Groq) use kar rahe ho, uski API key bhi Render/Railway ke **Environment Variables** mein daal dena — wahi jaisa `.env` mein hai. Koi extra setup nahi chahiye, kyunki wo already backend proxy ke through call ho rahi hai.

---

## Quick Summary — Deployment Order

1. Pehle GitHub pe poora code push karo (backend + frontend dono, `.env` files **NAHI** push karni — `.gitignore` mein hona chahiye)
2. Render/Railway pe backend deploy karo, env vars manually daalo
3. Vercel pe frontend deploy karo, `VITE_API_URL` env var backend ke live URL se set karo
4. CORS update karo backend mein
5. Test karo live URL pe jaake
